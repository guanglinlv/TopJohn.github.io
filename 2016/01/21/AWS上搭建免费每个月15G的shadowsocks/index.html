AWS上搭建免费每个月15G的shadowsocks

发表于 2016-01-21   |   分类于 shadowsocks   |  

前言

最早的时候，我是用GoAgent的，那个体验真的是差！差！差！，还得经常换ip，而且只能代理http，后来用免费的VPN，毕竟没给钱，太慢了，上了研究僧，用了导师的shadowsocks大半年，感觉很好，发现AWS的ec2可以免费用一年，那就试试吧。

注册AWS

注册Aws把我坑惨了，绑定信用卡，人家扣了1美元，我扣了2美元，可能是我绑的不是visa的卡的缘故，管它呢，反正会退，当电话验证的时候，我去，死活打不过来，后来打10086，我国际长途没开，可能是这个原因，卡了1天，后来用别人的手机验证的，验证完了，又扣了2美元（他大爷的），难道隔天又要重新验证嘛～～！！不过也不是大事，预授权的费用，大概3个星期就能回来，不虚。

创建EC2实例

我选的是Ubuntu Server 14.04 LTS，下面的全部默认，别乱选，万一超过免费范围了呢！选择东京的节点，据说效果比较好，然后配个ssh密钥。
打开终端，进到amazon.pem文件所在目录，
执行chmod 400 amazon.pem修改文件权限
执行ssh -i amazon.pem ubuntu@你的服务器节点ip即可连接上远程服务器
(ssh $user@$host -i $key 应该是限制了访问其他主机的时候都要用指定用户和指定的key文件)

安装shadowsocks依赖

  1. sudo -s //获取超级管理员权限
  2. apt-get update //更新apt-get
  3. apt-get install python-pip //安装python包管理工具pip
  4. pip install shadowsocks // 安装shadowsocks
  5. ssserver -c /etc/shadowsocks.json -d start //启动shadowsocks
    shadowsocks.json需要自己创建，默认是没有的

（ps. 可能有人不会用vi，这里简单说下，vi打开文件后，按i即可进入编辑状态，编辑完后，按esc退出编辑状态，按:进入命令状态，输入wq即可保存并退出，还不懂的话自行百度vi吧）

开启AWS入站端口

配置好shaodowsocks后，还需要将配置中的端口打开,这样客户端的服务才能链接得上EC2中的shadowsocks服务
首先打开正在运行的实例，向右滚动表格，最后一项，安全组，点击进入，编辑入站规则，默认是开启了一个22端口（这是给ssh访问的），再建一个如下图红框标示的端口，我的shadowsocks配置的端口是8388，所以这里就开启8388。

如果不放心流量超限的话可以设置下账单报警。
